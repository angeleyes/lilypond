Getting started

You've installed your shiny new LilyPond, and you can't wait to typeset music. Here are some hints.

Testing LilyPond

  • Open a terminal window (Windows users: double click the "GNU LilyPond" or "Cygwin" icon), and type 
         lilypond --help
    in your window. You should see something similar to 
         GNU LilyPond 1.8.0
     Usage: lilypond [OPTION]... FILE...
     
     Typeset music and or play MIDI from FILE
     
     LilyPond is a music typesetter.  It produces beautiful sheet music
     using a high level description file as input.  LilyPond is part of
     the GNU Project.
     
     Options:
       -h,--help             this help
     [..]

    If you see this, you have the program installed, and you can run it.

  • Open test.ly in your favorite text editor (Windows users: run notepad test.ly).
  • Enter 
         
     \score { \notes { c4-(  c4-) } }
    Save file and exit.
  • Run 
         ly2dvi -p test
    If everything works well, it should say 
         DVI output `test.dvi'...
     PS output to `test.ps'...
     PDF output to `test.pdf'...
  • View the PDF file. Try these commands 
       ggv test.pdf
   gv test.pdf
   kgv test.pdf
   acroread test.pdf
   ghostview test.pdf
    Windows users: go to folder C: -> CYGWIN -> HOME -> your-user-name, and double click test.pdf.

Yay, it works!

Darn, it doesn't work

Whoops, something went astray, so you have to send a bugreport. The directions for bugreports are in the FAQ. If your download was not packaged by the LilyPond team, CC your report to the packager listed in the "Courtesy" column on the download page

Windows users: run the cywgin terminal (double click the Cygwin or LilyPond icon), and run 

   sh -x /usr/bin/bug-lilypond-cygwin >& bug.txt
Attach the resulting bug.txt file to your bug-report.
