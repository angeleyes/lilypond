Brilliant hackers

We want our patches cooked as follows:
  • Follow the coding standards..
  • No complete files, only diffs
  • Unified diff (diff -u).
  • ChangeLog entries
  • patches for Documentation/topdocs/AUTHORS.texi and THANKS, if appropriate
Our entry point is in main() (lily/main.cc). Good luck.

“But I am not brilliant!&rdquo

There is lots to do, and not all of it requires programming. For example, you could
  • help translating. See http://www2.iro.umontreal.ca/~pinard/po/registry.cgi?domain=lilypond
  • help proofreading and editing.
  • help packaging.
  • help entering music for the Mutopia project.
  • pick out something from our TODO list. Read it by running 
       grep TODO `find`
    on the source archive.
  • jazz up our website. It is available from CVS: run 
      cvs -d:pserver:anoncvs@subversions.gnu.org:/cvsroot/lilypond login
  cvs -z3 -d:pserver:anoncvs@subversions.gnu.org:/cvsroot/lilypond co newweb
