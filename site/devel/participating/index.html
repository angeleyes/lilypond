Brilliant hackers

We want our patches cooked as follows:
  • Follow the coding standards..
  • No complete files, only diffs
  • Unified diff (diff -u).
  • ChangeLog entries
  • patches for Documentation/topdocs/AUTHORS.texi and THANKS, if appropriate
Our entry point is in main() (lily/main.cc). Good luck.

Less brilliant hackers

Not all hope is lost if you are not brilliant. There are other tasks open:
  • Help translating. See http://www2.iro.umontreal.ca/~pinard/po/registry.cgi?domain=lilypond
  • Pick out something from our TODO list
  • Help Packaging.
