Getting started

You've installed your shiny new LilyPond, and you can't wait to typeset music. Here are some hints.

Testing LilyPond

These are the instruction for Unix and Windows on the command line. Windows users are advised to a look at the quickstart method for Windows.

  • Open a terminal window and type 
    	lilypond --help
    in your window. You should see something similar to 
    	Usage: lilypond [OPTION]... FILE...

	Typeset music and/or produce MIDI from FILE.

	LilyPond produces beautiful music notation.
	For more information, see http://lilypond.org

	Options:
	-b, --backend=EXT     select backend to use
	-e, --evaluate=EXPR   set options, use -e '(ly:option-usage)' for help
	-f, --formats=EXTs    list of formats to dump
	-h, --help            print this help
	[..]

    If you see this, you have the program installed, and you can run it.

  • Start your favorite text editor.
  • Enter 
    	{
	c4( c)
	}
    NOTE: users of LilyPond 2.2 and earlier enclose the above example in \notes{} and \score {} blocks.
  • Save it as test.ly. Exit the editor.
  • Run 
    	lilypond test
    If everything works well, it should say 
    	DVI output `test.dvi'...
	PS output to `test.ps'...
	PDF output to `test.pdf'...
  • View the PDF file. Try these commands 
    	ggv test.pdf
	gv test.pdf
	kgv test.pdf
	acroread test.pdf
	xpdf test.pdf
	ghostview test.pdf
  • The easiest option for Windows users is the drag & drop approach. If you prefer the keyboard, you can try the above procedure, with the following notes:
    • double click the LilyPond icon
    • this will open a test file. Save it as test.ly on the desktop
    • double click test.ly
    • After a few seconds, a test.pdf file should appear.
